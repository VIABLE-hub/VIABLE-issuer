# BBS+ Docker Hot-Patching Guide

This guide explains how to use the hot-patching tool for BBS+ in Docker containers. This tool automatically fixes BBS+ issues in running containers without needing to rebuild or redeploy them.

## Overview

The StudentVC system uses BBS+ signatures for credential issuance and verification. When running in Docker containers, platform-specific differences in the BBS+ module can cause issues. The hot-patching tool automatically detects and fixes these issues in running containers.

## When to Use Hot-Patching

Use the hot-patching tool in the following scenarios:

1. **Production Emergencies** - When you need to fix BBS+ issues in production containers without downtime
2. **Attribute Name Mismatches** - When containers are using macOS-generated BBS+ bindings in Linux environments
3. **Missing .so Files** - When containers are missing the required .so files for BBS+ functionality
4. **UniFFI Contract Mismatches** - When containers have incompatible UniFFI contract versions

## Prerequisites

- Docker running with StudentVC containers deployed
- Access to the host machine running the containers
- Linux-compatible BBS+ files (generated by `test_bbs_linux_docker.sh`)

## Using the Hot-Patching Tool

### Quick Start

Run the hot-patching tool using:

```bash
make hot-patch-bbs
```

Or run the script directly:

```bash
./scripts/debugging/hot_patch_bbs_docker.sh
```

### How It Works

The hot-patching tool:

1. **Detects** running StudentVC containers
2. **Checks** if each container needs patching by testing BBS+ functionality
3. **Patches** containers by copying Linux-compatible BBS+ files
4. **Verifies** that the patch was successful by testing BBS+ functionality
5. **Restarts** containers if requested

### Patching Options

The tool provides two patching options:

1. **Patch All Containers** - Automatically patches all StudentVC containers
2. **Select Specific Containers** - Allows you to select which containers to patch

For each container, the tool:

1. Checks if BBS+ is working correctly with Linux attributes
2. If not, copies Linux-compatible BBS+ files to the container
3. Verifies that the patch was successful
4. Optionally restarts the container

## Automatic BBS+ File Generation

If Linux-compatible BBS+ files are not found in `backend/bbs-core/linux-build`, the tool will automatically generate them by running `test_bbs_linux_docker.sh`.

This ensures that you always have the correct files for patching, even if you haven't generated them manually.

## Production Deployment Workflow

For production deployments, follow this workflow:

1. **Test in Staging** - Run the hot-patching tool in a staging environment first
2. **Schedule Maintenance** - Schedule a maintenance window for production patching
3. **Backup** - Create backups of your containers before patching
4. **Patch** - Run the hot-patching tool on production containers
5. **Verify** - Verify that all containers are working correctly
6. **Update Images** - Update your Docker images to include the Linux-compatible BBS+ files for future deployments

## Troubleshooting

### Container Not Found

If the tool doesn't find any StudentVC containers, make sure they're running:

```bash
docker ps | grep studentvc
```

If no containers are running, start them:

```bash
make docker-run
```

### Failed to Generate Linux-Compatible BBS+ Files

If the tool fails to generate Linux-compatible BBS+ files:

1. Run `test_bbs_linux_docker.sh` manually
2. Check for errors in the output
3. Make sure you have the necessary permissions and dependencies

### Failed to Patch Container

If the tool fails to patch a container:

1. Check the container logs for errors
2. Use the debugging tool (`make debug-bbs-docker`) to investigate
3. Try restarting the container and patching again

## Conclusion

The BBS+ Docker hot-patching tool provides a quick and reliable way to fix BBS+ issues in running containers without needing to rebuild or redeploy them. By automatically detecting and fixing issues, it helps ensure that your StudentVC system remains operational even when facing platform-specific BBS+ challenges.
