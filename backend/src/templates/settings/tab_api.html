Quick Reference

Credential Management
  • • POST /api/v1/credentials/issue - Issue credential
  • • POST /api/v1/credentials/verify - Verify credential
  • • GET /api/v1/credentials/status/{id} - Check status
  • • GET /api/credentials - List all credentials
  • • POST /api/credential/{identifier}/revoke - Revoke a credential
  • • POST /api/credential/{identifier}/restore - Restore a revoked credential
  • • POST /api/credential/{identifier}/delete - Delete a credential
System & Health
  • • GET /api/v1/system/health - System health
  • • GET /api/system/health - Detailed health
  • • GET /settings/api/healthcheck - API healthcheck
  • • GET /health/system - System metrics
API Authentication

Generated API keys can be used to access all endpoints. Include your API key in the Authorization header: Authorization: Bearer YOUR_API_KEY
