Detaillierte Schlüsselverwendung

🔐

BBS+ Schlüssel

•
Verwendung: Signiert alle 30+ Felder der Studenten-Credentials
•
Prozess: Issuer → BBS+ Signatur → Wallet → ZK-Beweis → Verifier
•
Features: Selective Disclosure, Unlinkability, Zero-Knowledge
•
Beispiel: Student zeigt nur firstName, verbirgt lastName & studentId
🔏

JWT/Ed25519 Schlüssel

•
Verwendung: Signiert JWT-Envelope um gesamte VC
•
DID: did:key:zXwpQjZao...
•
Features: Issuer-Authentifizierung, Integrität, OpenID4VC
•
Verifikation: DID auflösen → Public Key extrahieren → JWT prüfen

🔄 Vollständiger Credential-Flow

1️⃣ Ausstellung
• Studentendaten erfassen
• Mit BBS+ signieren
• In JWT verpacken
• Mit Ed25519 signieren
2️⃣ Speicherung
• Wallet empfängt VC
• JWT validieren
• BBS+ Keys extrahieren
• Sicher speichern
3️⃣ Präsentation
• Felder auswählen
• ZK-Beweis erstellen
• Nur gewählte Daten
• An Verifier senden
4️⃣ Verifikation
• DID → JWT prüfen
• BBS+ Beweis prüfen
• Felder validieren
• ✅ Gültig/❌ Ungültig
🔑 Schlüsselrotation:
Alte Credentials bleiben mit alten Keys verifizierbar. Neue Credentials nutzen neue Keys.
🛡️ Sicherheit:
Private Keys nur auf Issuer-Server. Public Keys in DID für Verifier.
⚠️ Wichtig:
Jedes Deployment hat eigene Keys! TUB ≠ FUB ≠ ROOT
