Präsentationsempfang
Diese Prüfung analysiert, ob die Wallet eine gültige OpenID4VP-Präsentation übermittelt hat. Der QR-Code enthält eine Präsentationsanfrage, die von der Wallet verarbeitet und beantwortet wird.
⏳
Input: QR-Code Scan → OpenID4VP Request
Process: Warte auf vp_token Parameter via POST /direct_post
Validation: JWT-signiert + mandatory_fields Compliance
Output: presentation_received_status → success/error
Schlüssel extrahiert Issuer- und Holder-DIDs aufgelöst
?
Schlüsselextraktion
Aus den DID-Identifikatoren (iss, sub) werden die öffentlichen Schlüssel für die kryptographische Verifikation extrahiert. Diese Schlüssel ermöglichen die Prüfung der digitalen Signaturen.
⏳
Input: DID identifiers (iss, sub) from VP token
Process: did_to_key() → EC256 public key extraction
Algorithm: DID:key → Multicodec → Raw EC256 Point
Output: holder_public_key, issuer_public_key
Aussteller-Schlüssel geprüft Vertrauenswürdiger Issuer validiert
?
Aussteller-Verifikation
Der öffentliche Schlüssel des Ausstellers wird gegen eine Vertrauensliste geprüft, um nur autorisierte Institutionen zuzulassen. Dies stellt sicher, dass nur bekannte Bildungseinrichtungen Credentials ausstellen können.
⏳
Input: issuer_public_key from DID resolution
Process: issuer_key_is_valid() → whitelist validation
Whitelist: Trusted educational institutions registry
Output: issuer_trusted_status → boolean

Signatur verifiziert JWT ES256 kryptographisch geprüft
?
Signaturverifikation
Die ES256-Signatur des VP-Tokens wird kryptographisch verifiziert, um Integrität und Authentizität zu gewährleisten. Dies bestätigt, dass das Credential nicht manipuliert wurde.
⏳
Input: vp_token + holder_public_key
Algorithm: ES256 (ECDSA P-256 + SHA-256)
Process: jwt.decode(vp, holder_key, algorithms=["ES256"])
Output: signature_valid → boolean
BBS+ Schlüssel geprüft Selective Disclosure Signatur validiert
?
BBS+ Verifikation
Der BBS+ öffentliche Schlüssel ermöglicht Zero-Knowledge-Proofs und Selective Disclosure ohne Preisgabe aller Daten. Dadurch können nur relevante Informationen gezeigt werden.
⏳
Input: bbs_dpk from VC credential
Algorithm: BLS12-381 curve pairing verification
Process: DID:web resolution → BBS+ DPK validation
Output: selective_disclosure_enabled → boolean

Pflichtfeld-Validierung
Überprüfung, ob alle in der Präsentationsdefinition geforderten Felder vorhanden sind. Dies stellt sicher, dass das Credential vollständig ist.
⏳
Required Fields: iss, sub, total_messages, bbs_dpk
Mapping: Backend snake_case ↔ iOS camelCase
Process: Field presence validation + type checking
Output: mandatory_fields_complete → boolean
Credential-Gültigkeit Zeitliche Validität und Widerrufsstatus
?
Gültigkeitsprüfung
Prüfung der Gültigkeitsdauer und des Widerrufsstatus, um sicherzustellen, dass das Credential noch aktiv ist und nicht widerrufen wurde.
⏳
Input: expirationDate + validity_identifier
Process: Timestamp validation + revocation list check
Registry: Distributed revocation registry query
Output: credential_valid + not_revoked → boolean
